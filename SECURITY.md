# Security Policy

This document explains which versions receive security updates and how to report vulnerabilities responsibly.

## Supported versions

We provide security fixes for:

- The `main` branch
- The latest release

Older releases may not receive fixes. If you are running an older version, please upgrade before reporting.

## Reporting a vulnerability

Do not report security vulnerabilities in public GitHub issues.

### Preferred channel

Open a private GitHub Security Advisory for this repository.

### Alternate channel

If advisories are unavailable, email **abhishek@convos.store** with `[SECURITY]` in the subject line.

### What to include

Please include:

- A clear description of the vulnerability
- Affected routes, files, or components
- Steps to reproduce or proof of concept
- Impact assessment (what could happen if exploited)
- Any suggested mitigation if known

### What happens next

We will:

1. Acknowledge your report promptly
2. Investigate and validate the issue
3. Share remediation updates until closure

## Coordinated disclosure

We appreciate coordinated disclosure and will work with reporters to agree on timelines when possible. Please allow time for verification and patching before public disclosure.
